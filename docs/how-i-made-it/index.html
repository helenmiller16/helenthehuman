How I made it

01 Apr, 2021

I made a site and published it. The proof is that you’re reading this. I’ve been meaning to do this for a while because I’ve heard that having a personal website can be useful, and it turned out to be not too hard. Here’s what I did:

  1. I chose Hugo as a static site generator, because that’s what blogdown uses, which I might decide to use some day. If I don’t, I might change to jekyll, since it has some nice integrations with github pages.

  2. Choosing a theme took the most time. I chose this super simple bearblog theme because it was easy to get started with, and I figured it would probably be easy to customize or port my stuff to a new theme if I decide to do so.

  3. Using GoDaddy, I purchased helenthehuman.com. I’m working on my personal branding, and figured helenthehuman is a good option becuase I want to distinguish myself from those other Helens out there. I used GoDaddy because I had heard of it before and didn’t know of any other DNS providers out there…

  4. Now that I had made all the hard decisions, I set up my project, using the instructions from Hugo, and pushed it to a new repo on github. I set it up as a project site, because you only get one user site, and I’m afraid of commitment.

  5. It seemed like the path of least resistance was to build my site myself and commit the changes. Github by default only allows you to publish a site from the docs subfolder (there’s probably a way around that, but I didn’t bother to figure it out). So, I added this line in my config.toml so that it builds the site into docs (as opposed to the default public):

    publishDir = "docs"

  6. The final step was to get github pages running and point my domain to wherever github was serving it. These instructions worked great.

You can see all the source code at https://github.com/helenmiller16/helenthehuman.
